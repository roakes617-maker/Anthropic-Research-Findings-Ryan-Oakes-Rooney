Cowork Feedback
Cowork in the Enterprise
Enterprise perspective after hands-on testing — requested by Lauren ahead of our next discussion.

Core thesis: The technology is ready. The barrier to transformative success is enterprise planning, design, standardization, and change management.

⚠️  Enterprise Deployment Strategy
Without Intent
✗ Majority of users treat it as a conversational agent
✗ Low adoption of plugins, skills, and commands
✗ Minimal workflow change
✗ The ROI story dies
With Intent
✓ Design plugins, skills, tools around jobs-to-be-done (outcomes, not features)
✓ Prioritize: highest impact / lowest complexity
✓ Pair deployment with training & change management
✓ Dramatically larger ROI — chatbot → workforce multiplier

Where Agentforce Falls Short
✗ Completely operationally built & managed — no end-user customization
✗ Users can't create their own skills, commands, or tools
✗ No web search from day one — missed the #1 adopted AI feature
But Salesforce Sees This — Enter Slackbot
→ Salesforce leaning hard into Slackbot to address these gaps
→ Not grounded in web, but rich enterprise context — channels, conversations, shared knowledge
→ Returns contextual, web-like responses users find valuable
→ Slack team now building "skills" — mimicking Cowork's design pattern
→ I see Slackbot as a more direct competitor to Cowork than anything else
The differentiator: Slackbot wins on convenience (users already live in Slack). Cowork wins on enterprise architecture (if IT/Ops deploys with jobs-to-be-done design + change management).
🚀  Where Cowork Takes Off — Ambient, Always-On Agents

Cowork takes off when it isn't reliant on being user-initiated.

Why OpenClaw Went Viral
→ Ambient, always-on architecture
→ Works on your behalf, not just by your side
→ Early adopters want agents that don't wait to be asked
The Security Reality
→ OpenClaw (now OpenAI) — massive security concerns
→ Enterprises actively banning this tool
→ But the signal is clear: market wants always-on agents
🔥  Best Example: Regrello on Agentforce
→ Triggered when email arrives (e.g. change order from customer to Flex)
→ Agent kicks off series of workflows to complete the entire job
→ Loops in human at review/approve steps
→ Created in minutes — feels fully agentic
→ A pattern Cowork could adopt — ambient, trigger-based agents with safety-first design
The Opportunity

Architecture is right. Product is strong. The gap is between "deploy" and "transform" — that's a change management problem, not a technology problem. Enterprises that treat Cowork as a platform to design around (not just a tool to turn on) will see 10x the results. That's exactly the conversation I'd be having with every customer.

Independent research across models, products, and enterprise positioning. Updated February 2026.

Claude Cowork & Code vs. OpenClaw — Fundamental Differences
OpenClaw (formerly Clawdbot/Moltbot) went viral in Jan 2026 as an open-source personal agent framework. Creator Peter Steinberger joined OpenAI on Feb 15. All three tools emerged from the same model breakthrough — frontier models capable of autonomous action — but they're built for fundamentally different audiences with opposite design philosophies. Cowork and Code are enterprise-grade products with managed security, predictable pricing, and vendor support. OpenClaw is an open-source framework that maximizes power and flexibility at the cost of enterprise readiness.
Claude Cowork
For: knowledge workers (non-technical)
How: desktop app · sandboxed to user-granted folders · 11 open-source plugins across sales, marketing, legal, finance, support, PM · skills & connectors for standardized workflows
Architecture: Apple Virtualization Framework sandbox · destructive actions require confirmation · no persistent daemon · tasks are discrete, auditable, reset between sessions
Enterprise edge: managed by Anthropic · flat $100-200/mo subscription · no data leaves trust boundary · enterprise sales + AWS/GCP partnerships · security teams approve it
Market impact: built by Claude Code in ~1.5 weeks · S&P Software & Services Index down 20% YTD — market pricing in enterprise disruption from a vendor with real GTM, not just a viral repo
Claude Code
For: developers & engineers
How: terminal-based · reads codebase, writes code, runs tests, manages git · VS Code, JetBrains, Xcode integration
Architecture: session-based · scoped to codebase · no persistent system access · operates through existing dev toolchain
Enterprise edge: 4% of all public GitHub commits and growing (projected 20%+ by EOY) · est. $1-2B annualized revenue faster than ChatGPT · 77.2% SWE-bench
Real example: my SEs used Claude Code to build custom Salesforce LWCs for an Uber demo under a tight deadline — hit the deadline, deepened the exec relationship
OpenClaw (now OpenAI)
For: power users, hobbyists, prosumers
How: open-source · self-hosted · runs 24/7 as persistent daemon via messaging apps (WhatsApp, Slack, iMessage) · full system access · 180K+ GitHub stars
Architecture: heartbeat architecture (proactively wakes, reviews context, takes action) · CLI-first, rejects MCP in favor of --help discovery · model-agnostic (works with DeepSeek, Claude, GPT)
Enterprise risk: 17-20% malicious skills on ClawHub · Bitdefender recommends not running on company devices · no vendor support · unpredictable API costs ($300-750/mo) · Korean tech firms banning it
Bottom line: viral consumer adoption (Baidu integrating it) but enterprise security teams are actively blocking it — Steinberger joining OpenAI signals pivot from open-source to OpenAI's Frontier platform
OpenAI Codex
For: developers & engineers
How: macOS desktop app (Feb 2026) + CLI + IDE extension · cloud sandboxed · multi-agent parallel workflows · GPT-5.3-Codex model
Architecture: cloud-based execution (tasks run in OpenAI sandbox, not locally) · Git worktree isolation · async delegation model — more like assigning tasks to a colleague than pair programming
vs. Claude Code: Codex runs in OpenAI's cloud → code leaves your environment · Claude Code runs locally in your terminal → code stays on your machine · Codex optimized for async delegation, Claude Code for real-time pairing
Key difference: OpenAI acquired OpenClaw creator to "drive the next generation of personal agents" + launched Frontier platform for enterprise agent management — consolidating consumer + enterprise agent strategy under one roof
Sales angle: Four tools, two philosophies. Anthropic's Cowork + Code are enterprise-grade: sandboxed, locally executed, vendor-supported, predictable pricing. OpenClaw maximizes power but fails enterprise security review. Codex runs in OpenAI's cloud — your code leaves your environment. When a CIO asks "why not Codex?" → your code stays local with Claude Code. "Why not OpenClaw?" → security, support, and predictable cost.
